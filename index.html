Tarea sobre Desarrollo de Software

1. Qué son los Modelos de Estimación?

Los modelos de estimación en el desarrollo de software son métodos o técnicas utilizadas para predecir el esfuerzo, el tiempo y los recursos necesarios para completar un proyecto de software.

2. Modelos de Estimación en la Industria del Desarrollo de Software

Modelo COCOMO (Constructive Cost Model): Este modelo se utiliza para estimar el esfuerzo y el costo de desarrollo de software, considerando diferentes niveles de complejidad y factores adicionales como la reutilización de software.

Método de Puntos de Función: Mide el tamaño funcional de un sistema basado en las funcionalidades proporcionadas al usuario y su complejidad funcional.

Comparación y Contraste: COCOMO se enfoca en el tamaño del código y la complejidad general del proyecto, mientras que Puntos de Función se centra en las funcionalidades del sistema y su complejidad funcional. COCOMO es más adecuado cuando se tiene una buena especificación de requisitos, mientras que Puntos de Función es útil cuando los requisitos funcionales están bien definidos.

3. Estimación de un Proyecto de Software

La estimación de un proyecto de software implica predecir el esfuerzo, tiempo y recursos necesarios para su desarrollo, facilitando la planificación y gestión del proyecto.

4. Principios Esenciales para el Desarrollo de Sistemas

  • Modularidad: Divide el sistema en módulos independientes para facilitar el desarrollo y mantenimiento.
  • Cohesión: Cada módulo debe realizar una tarea específica y bien definida.
  • Acoplamiento: Minimiza la dependencia entre módulos para facilitar cambios y mejoras.
  • Abstracción: Oculta detalles complejos y proporciona interfaces claras para interactuar con el sistema.

Ejemplo de Aplicación - Modularidad: En el desarrollo de un sistema de gestión de inventario, se pueden crear módulos para funciones como gestión de productos, control de stock y gestión de pedidos, facilitando la asignación de tareas y mejorando la mantenibilidad del sistema.

5. Partes Interesadas en un Proyecto de Desarrollo de Software para una Aplicación Móvil

  • Cliente/Usuario final: Define requisitos, prueba el producto final.
  • Equipo de Desarrollo: Desarrolla y prueba el software según especificaciones.
  • Gestor de Proyecto: Coordina el proyecto, gestiona recursos y plazos.
  • Departamento de TI/Operaciones: Implementa y mantiene el sistema, proporciona soporte técnico.

6. Diseño de un Plan de Proyecto para un Sistema de Gestión de Inventario

  1. Fase 1: Análisis y Requisitos
    • Actividades: Definición de requisitos, análisis de viabilidad.
    • Dependencias: Ninguna.
  2. Fase 2: Diseño del Sistema
    • Actividades: Diseño arquitectónico, diseño detallado.
    • Dependencias: Requisitos definidos en la fase anterior.
  3. Fase 3: Desarrollo y Codificación
    • Actividades: Codificación de módulos, pruebas unitarias.
    • Dependencias: Diseño del sistema completado.
  4. Fase 4: Pruebas y Validación
    • Actividades: Pruebas de integración, pruebas de aceptación.
    • Dependencias: Desarrollo y codificación finalizados.
  5. Fase 5: Implementación y Mantenimiento
    • Actividades: Implementación del sistema, soporte post-implementación.
    • Dependencias: Pruebas y validación exitosas.

7. Normas y Estándares en el Desarrollo de Software

  • Consistencia en la Codificación: Establecer convenciones de nombres, indentación y estructura de código.
  • Documentación Detallada: Documentar funciones, clases y procesos utilizando comentarios claros y consistentes.
  • Ejemplo de Aplicación: En un proyecto, se podría establecer una norma donde todos los nombres de variables sigan la convención camelCase y se documenten las funciones utilizando Javadoc o comentarios similares para mejorar la legibilidad y mantenibilidad del código.
